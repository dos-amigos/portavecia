# Phase 3 — UI Design Contract

> Visual and interaction contract for Gallery and Events pages. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Kirby CMS / PHP — shadcn not applicable) |
| Preset | not applicable |
| Component library | none (server-rendered PHP templates + Alpine.js) |
| Icon library | none (Unicode emoji for wine glass, inline SVG for arrows) |
| Font heading | Playfair Display (self-hosted via @fontsource), 400 + 700 |
| Font body | Raleway (self-hosted via @fontsource), 400 + 700 |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Masonry gap on mobile, inline icon padding |
| sm | 8px | Filter button internal padding, photo counter spacing |
| md | 16px | Gallery grid gap, event card internal padding |
| lg | 24px | Event card content padding (p-6), section title margin-bottom |
| xl | 32px | Gap between filter bar and grid, gap between event sections |
| 2xl | 48px | Section breaks between "Prossimi Eventi" and "Eventi Passati" |
| 3xl | 64px | Page-level vertical padding (section-padding: py-16 md:py-24) |

Exceptions: Gallery masonry gap uses 12px (gap-3) for tighter visual rhythm in photo grids. This is 3x4px, still on the 4px sub-grid.

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 14px (text-sm) | 400 | 1.5 | Raleway |
| Label | 12px (text-xs) | 700 | 1.4 | Raleway |
| Heading | 20px (text-xl) | 400 | 1.2 | Playfair Display |
| Display | 28px (text-2xl md:text-3xl) | 700 | 1.2 | Playfair Display |

**Weights used: 2 only — 400 (regular) and 700 (bold).** No intermediate weights (600/semibold) are permitted in this phase.

Usage in this phase:
- **Display:** Page titles ("Galleria", "Eventi")
- **Heading:** Event card titles, lightbox captions
- **Body:** Event descriptions, photo captions, filter button labels
- **Label:** Event date/time metadata, photo counter ("3 / 12"), category tags, active filter labels, "Eventi Passati" badge

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(0.15 0 0)` (--color-dark) | Page background, gallery backdrop, lightbox overlay |

| Text primary | `oklch(0.96 0.01 80)` (--color-light) | All headings, body text, active filter label |
| Text muted | `text-light/70` | Event descriptions, photo captions, inactive filter labels |

| Destructive | not needed | No destructive actions in this phase |

Accent reserved for: active filter tab background, event date display, "Prenota su WhatsApp" CTA on event cards, lightbox navigation highlight, photo counter number.

## Focal Point Declaration

Each page template declares a single primary visual anchor that draws the visitor's eye first.

| Template | Focal Point | Rationale |
|----------|-------------|-----------|
| Gallery | Masonry photo grid filling the viewport below the filter bar | The photos are the content. The filter bar is secondary navigation; the grid itself is the immediate draw. Dense visual mass of images creates strong focal pull. |
| Events | Upcoming event card(s) with accent-colored date badge | The date badge in accent color breaks the neutral palette and signals "this is next." Past events are dimmed to 60% opacity, further directing attention to upcoming cards. |

## Interaction Contract

### Gallery Filter

| Interaction | Implementation | Timing |
|-------------|---------------|--------|
| Click filter button | Alpine.js toggles `activeFilter` state variable | Instant state change |
| Photo visibility | `x-show="activeFilter === 'all' || category === activeFilter"` on each photo | 200ms fade transition via `x-transition:enter="transition ease-out duration-200"` |
| Masonry reflow | CSS columns handle reflow automatically when items hide/show | Native browser reflow |
| Filter URL state | No URL persistence needed -- page-level only | N/A |

### Gallery Lightbox

| Interaction | Implementation | Timing |
|-------------|---------------|--------|
| Open lightbox | Click on photo triggers GLightbox open | GLightbox default (~300ms) |
| Navigate photos | Swipe (mobile) or arrow keys/buttons (desktop) | GLightbox default |
| Photo counter | GLightbox `slideCount` option | Displays "3 / 12" format |
| Filter-aware cycle | Dynamically build GLightbox elements array from visible (filtered) photos only | Rebuild on filter change |
| Close lightbox | Click overlay, X button, or Escape key | GLightbox default |

### Events

| Interaction | Implementation | Timing |
|-------------|---------------|--------|
| Prenota CTA click | Opens WhatsApp with pre-filled message including event name | Instant redirect |
| Past/future sort | Server-side via Kirby: compare event date to `date('Y-m-d')` | On page load |
